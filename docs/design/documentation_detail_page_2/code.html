챗봇 서비스 배포하기

Choorai의 관리형 인프라를 사용하여 몇 분 만에 AI 기반 챗봇 서비스를 구축, 구성 및 배포하는 방법을 알아보세요.

Choorai를 사용하면 웹 서비스를 손쉽게 배포할 수 있습니다. 이 가이드에서는 챗봇을 실행하고 운영하는 데 필요한 필수 단계를 다룹니다. 사전 구축된 모델을 활용하여 서버 관리 대신 대화 로직에 집중할 수 있습니다.

info
꿀팁

Choorai AI는 모든 기본 인프라 확장을 처리합니다. 대화 로직과 구성 매개변수만 제공하면 됩니다.

사전 요구 사항

  • Choorai 계정: 결제가 활성화된 활성 계정이 있는지 확인하세요.
  • API 키: 대시보드 설정에서 프로젝트 API 키를 생성하세요.
  • CLI 도구: 최신 버전의 Choorai CLI를 설치하세요.
터미널 
npm install -g @choorai/cli
choorai init my-chatbot

구성 설정

choorai.config.json 파일에서 서비스 매개변수를 정의합니다. 여기서 모델 유형, 메모리 할당 및 사용자 지정 동작 프롬프트를 지정할 수 있습니다.

warning
보안 주의

구성 파일에 API 키를 직접 커밋하지 마세요. 항상 환경 변수 또는 시크릿 매니저를 사용하세요.

choorai.config.json 
{
  "name": "customer-support-bot",
  "model": "gpt-4-turbo",
  "memory": 512,
  "env": {
    "OPENAI_API_KEY": "${SECRETS.OPENAI_KEY}"
  }
}

배포하기

구성이 완료되면 단일 명령으로 배포할 수 있습니다. 시스템이 구성을 패키징하고 자격 증명을 검증한 후 확장 가능한 엔드포인트를 프로비저닝합니다.

터미널 
choorai deploy --prod
check_circle
배포 성공

서비스가 라이브 상태입니다! Choorai 대시보드에서 분석 및 로그를 확인할 수 있습니다.

서비스 테스트하기

`curl` 또는 내장된 테스트 클라이언트를 사용하여 배포된 챗봇을 즉시 테스트할 수 있습니다. 

curl -X POST https://api.choorai.app/v1/chat \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"message": "안녕하세요, 비밀번호를 어떻게 재설정하나요?"}'
