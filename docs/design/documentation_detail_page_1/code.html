How to Deploy a Chatbot Service

Learn how to build, configure, and deploy an AI-powered chatbot service in minutes using Choorai's managed infrastructure.

Choorai makes it effortless to deploy web services. This guide covers the essential steps to get your chatbot up and running. By leveraging our pre-built models, you can focus on the conversation logic rather than managing servers.

info
Managed Infrastructure

Choorai AI handles all the underlying infrastructure scaling. You just provide the conversation logic and configuration parameters.

Prerequisites

  • Choorai Account: Ensure you have an active account with billing enabled.
  • API Keys: Generate your project API keys from the dashboard settings.
  • CLI Tool: Install the latest version of Choorai CLI.
Terminal 
npm install -g @choorai/cli
choorai init my-chatbot

Configuration

The choorai.config.json file is where you define your service parameters. You can specify the model type, memory allocation, and custom behavior prompts here.

warning
Security Notice

Never commit your API keys directly into the configuration file. Always use Environment Variables or the Secrets Manager.

choorai.config.json 
{
  "name": "customer-support-bot",
  "model": "gpt-4-turbo",
  "memory": 512,
  "env": {
    "OPENAI_API_KEY": "${SECRETS.OPENAI_KEY}"
  }
}

Deployment

Once configured, deployment is a single command. The system will package your configuration, validate your credentials, and provision a scalable endpoint. 

choorai deploy --prod
check_circle
Deployment Successful

Your service is live! You can now view analytics and logs in the Choorai Dashboard.

Testing Your Service

You can test your deployed chatbot immediately using `curl` or our built-in test client. 

curl -X POST https://api.choorai.app/v1/chat \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"message": "Hello, how can I reset my password?"}'
