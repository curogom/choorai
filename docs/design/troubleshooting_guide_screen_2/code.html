문제 해결 & 솔루션

일반적인 빌드 및 배포 오류에 대한 해결책을 즉시 찾아보세요.

자주 찾는 에러:
심각 코드: 403

Access to XMLHttpRequest blocked by CORS policy

info 원인

프론트엔드가 다른 도메인에서 호스팅되는 백엔드로부터 데이터를 요청하려고 했지만, 브라우저가 보안상의 이유로 이를 차단했습니다.

check_circle 해결책
// 백엔드 설정에서
Access-Control-Allow-Origin: "https://your-frontend.com"

또는 설정 > 보안 > 허용된 오리진으로 이동하여 도메인을 추가하세요.

주의 런타임

process.env.API_KEY is undefined

info 원인

애플리케이션이 `API_KEY` 환경 변수를 읽을 수 없습니다. 이는 일반적으로 클라우드 대시보드에 변수가 설정되지 않았을 때 발생합니다.

check_circle 해결책
  1. 프로젝트 대시보드로 이동하세요.
  2. 설정 > 환경 변수로 이동하세요.
  3. 키 추가: API_KEY와 값을 입력하세요.
  4. 애플리케이션을 다시 배포하세요.
심각 빌드

Error: Command "npm run build" exited with 1

info 원인

빌드 스크립트가 실패했습니다. 이는 종종 코드의 구문 오류, 누락된 종속성 또는 TypeScript의 타입 오류로 인해 발생합니다.

check_circle 해결책
1단계: 이 오류 줄 바로 위의 로그를 확인하세요.
2단계: "Module not found" 또는 "SyntaxError"를 찾으세요.
3단계: 로컬에서 npm install을 실행하여 종속성이 올바른지 확인하세요.

에러를 찾을 수 없나요?
