## 핵심 문제: 학습 경로 UX의 불명확함

### 현재 구조
```
홈페이지 (/)
├── 시작하기
│   └── 60분 완주 (React + FastAPI) ← MVP, 빠른 시작
├── 📚 학습 로드맵
│   ├── Level 0: 도구 준비 ✅
│   └── Level 1-3: 기초 학습 🔒 (준비 중)
└── 🚀 서비스 사이클
    ├── Cycle 1-6: DNS → Ops ← 심화, 운영 수준
    ├── 프레임워크 옵션 (Vue, Hono, Go...)
    └── 인증 가이드
```

### 핵심 문제점

| 문제 | 현재 상태 | 사용자 혼란 |
|------|----------|------------|
| **1. 진입점 불명확** | 홈에서 어떤 경로로 가야 할지 선택지 없음 | "뭘 먼저 해야 하지?" |
| **2. 60분↔사이클 관계 모호** | 0-overview에만 "60분 후 추천" 언급 | "둘 중 하나만 하면 되나?" |
| **3. 용어 혼동** | Level/Cycle/학습/서비스 사이클 | "다 같은 건가?" |
| **4. 네비게이션 계층** | 화살표(→)로만 하위 항목 구분 | "인증도 Cycle 7인가?" |
| **5. 역방향 참조 없음** | 사이클 → 60분 완주 링크 없음 | 경로 관계 역방향 이해 불가 |

---

## 개선 방향

### 철학: "학습 경로 vs 레퍼런스 분리"

**사용자 유형 인식**:
- **초보자**: 60분 완주로 첫 배포 경험 → 이후 필요한 것 추가
- **경험자**: 특정 가이드만 참고 (예: Hono 배포, Supabase 연결)
- **참조 목적**: 트러블슈팅, 특정 에러 해결

```
┌─────────────────────────────────────────────────┐
│ 📚 학습 경로 (시작하기)                          │
│   └── 60분 완주: 초보자를 위한 순차적 가이드     │
├─────────────────────────────────────────────────┤
│ 📖 레퍼런스 (서비스 사이클)                      │
│   ├── DNS 가이드: 도메인 연결이 필요할 때        │
│   ├── Frontend 가이드: 프론트 배포가 필요할 때   │
│   ├── Backend 가이드: API 배포가 필요할 때       │
│   ├── Database 가이드: DB 연결이 필요할 때       │
│   ├── 인증 가이드: 로그인이 필요할 때            │
│   └── ...                                       │
└─────────────────────────────────────────────────┘
```

**핵심 원칙**:
1. **60분 완주** = 순차적 학습 경로 (초보자 전용)
2. **서비스 사이클** = 독립적 레퍼런스 (필요한 것만 참고)
3. **순차 강조 줄이기**: "Cycle 1 → 2 → 3" 대신 "필요한 가이드 선택"

## Phase 3: 네비게이션 구조 개선

### 현재 문제
```
🚀 서비스 사이클:
  - Cycle 1: DNS
  - Cycle 2: Frontend
  - → Vue (Lv.2)         ← 계층 불명확
  - Cycle 3: Backend
  - → Hono (Lv.1)
  - → Go (Lv.3)
  - 인증 가이드          ← Cycle 7처럼 보임
```

### 개선안: 카테고리별 레퍼런스 구조

**파일**: `site/src/layouts/DocsLayout.astro`

```

🌐 도메인
├── DNS 설정

├── React

├── FastAPI
├── Hono                      [추천]
├── Go
└── NestJS

🗄️ 데이터베이스               [▼]

🔐 인증                        [▼]
├── 개요

└── ...

⚙️ 운영
├── 환경 변수
├── 모니터링
```

### 핵심 변화
1. **"Cycle 1, 2, 3..." 순차 번호 제거** → 카테고리별 분류
2. **접을 수 있는 섹션**: 프레임워크 옵션 많은 카테고리
3. **레벨 배지 대신 추천 배지**: 복잡도보다 "추천" 강조
4. **트러블슈팅 통합**: 레퍼런스 느낌 강화

## Phase 4: 용어 및 URL 구조 정리

### 현재 혼동
- "서비스 사이클"이라는 용어가 순차적 학습을 암시
- `/cycle/` URL이 순차적 느낌

### 개선안

| 기존 | 개선 | 비고 |
|------|------|------|
| 서비스 사이클 | **가이드** | 레퍼런스 느낌 강화 |
| Cycle 1-6 | 카테고리명 (도메인, 프론트엔드...) | 번호 제거 |
| 학습 로드맵 | 유지 (준비 중) | - |

### URL 구조 (기존 유지, 리다이렉트 가능)
- `/cycle/1-dns` → `/guide/dns` (선택적 마이그레이션)
- 또는 기존 URL 유지하되 네비게이션만 변경

## 검증 방법

1. **개발 서버**: `cd site && npm run dev`
2. **홈페이지**: "처음이에요" / "필요한 것만 찾아요" 진입점 확인
3. **0-overview**: 가이드 인덱스 기능 확인
4. **네비게이션**: 카테고리별 분류, 접기/펼치기 확인
5. **모바일**: 햄버거 메뉴 동작 확인

## 기대 결과

### Before
- "60분 완주와 사이클, 뭘 먼저 해야 하지?"
- "Hono 가이드만 보고 싶은데 어디있지?"
- "인증도 필수로 해야 하나?"

### After
- **초보자**: 홈에서 "처음이에요" → 60분 완주
- **경험자**: 홈에서 "필요한 것만" → 가이드 인덱스 → 원하는 가이드
- 네비게이션에서 **카테고리별로 정리**되어 원하는 것 바로 찾기
- **순차 강제 없음**: 필요한 가이드만 참고 가능
